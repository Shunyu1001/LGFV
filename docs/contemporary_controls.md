# Contemporary Controls

This note documents the control-variable collection files generated by `scripts/build_contemporary_control_queue.py`.

## Outputs

- `data/analysis_inputs/contemporary_city_controls.csv`
- `data/analysis_inputs/contemporary_city_controls_source_backed.csv`
- `data/analysis_inputs/platform_control_coding_queue.csv`

The city-control file currently has 64 control units built from the 94 gold-standard cases. District or county platforms are assigned to their matched prefecture-level control unit when the historical-capacity crosswalk supplies a prefecture match.

The platform-control queue currently has 94 gold-standard case rows. Platform administrative level is pre-coded only when a conservative rule can infer it from the platform name, specialized sector, or from a city-prefecture mismatch. The current rules populate 82 administrative-level pre-codes. These rule-based values remain pending human check.

## Control Definitions

- Fiscal self-sufficiency: general public budget revenue divided by general public budget expenditure.
- Debt pressure: local government debt balance divided by general public budget revenue.
- Land finance dependence: land conveyance revenue divided by general public budget revenue.
- GDP per capita: GDP divided by resident population, preferably from the same official statistical communique.

Preferred year is 2024, with 2023 as fallback. Values should come from official city statistical communiques, budget final accounts, local debt disclosures, or finance-bureau final-account tables. When the first-pass source is a rating-agency compilation of public data, the status field marks it as `source_backed_secondary_compilation` so it can later be replaced by official source tables. The templates include Chinese search queries for each control unit to make source collection reproducible.
